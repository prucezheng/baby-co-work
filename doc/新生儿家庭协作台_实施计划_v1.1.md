# 新生儿家庭协作台 v1.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 4 人 / 40 小时内交付可远程体验的移动端 Web App，完成“成员身份 → 文字/语音创建 → 知识增强 → 自动分配并直接发布 → AI 分解 → 本人完成 → 即时感谢与贡献反馈”主闭环。

**Architecture:** 保留现有 React + Vite 前端、Express 服务端、Zod 共享契约和 Ark 适配器；新增单进程 JSON 持久化仓储、会话鉴权、任务编排服务与事件派生统计。文字和语音转写统一进入 `TaskOrchestrator`；参考视频只进入附件服务，明确禁止调用任务生成模型。生产环境由同一个 Node 进程提供 API、受控媒体下载和 Vite 静态资源，`DATA_DIR` 指向可持久化目录。

**Tech Stack:** TypeScript, React 19, Vite, React Router, Zustand, Express 5, Zod 4, Vitest, Testing Library, Playwright, Node `crypto/fs`, Multer, Python Ark Audio Transcriber, ffmpeg, Doubao Seed 2.0 Lite (`doubao-seed-2-0-lite-260428`).

## 0. 实现基线、边界与交付结构

唯一需求基线是 `doc/新生儿家庭协作台_PRD_v1.1.md`。以下历史资产只用于视觉或技术参考，不能覆盖 v1.1：

- `doc/新生儿家庭协作台_PRD_v1.0.md`
- `doc/新生儿家庭协作台_实施计划_v1.0.md`
- `功能板块设计交接清单.md`
- 旧 `/api/analyze` 视频分析实验链路

必须遵守的产品边界：

1. 底部导航固定为“今日 / 创建 / 贡献 / 家庭”。
2. 文字和最长 60 秒语音进入同一个任务 Schema；成功后直接发布，不经过确认页。
3. 视频只作为父任务的“怎么完成”附件，上传视频不得触发 Ark 任务生成或子任务生成。
4. 知识库只补充步骤、遗漏和提示，不改变原始意图，不主动创建日程。
5. 不展示风险等级、排行、倒数、扣分或积分商城。
6. 诊断、剂量、停药和紧急症状输入仍可保存原始任务，但模型不得扩写医疗步骤，只添加非阻断式就医提示。
7. 贡献从不可变完成事件派生；子步骤不单独计贡献。

| Code | HTTP | 用户动作 |
|---|---:|---|

| `UNAUTHENTICATED` | 401 | 重新输入 PIN |
| `FORBIDDEN` | 403 | 返回可执行页面 |
| `PIN_LOCKED` | 423 | 显示剩余锁定时间 |
| `NOT_FOUND` | 404 | 刷新列表 |
| `VERSION_CONFLICT` | 409 | 刷新后重试 |
| `DUPLICATE_REQUEST` | 200 | 返回原任务而非新建 |
| `TRANSCRIPTION_FAILED` | 422 | 保留录音并重试/改文字 |
| `ARK_UNAVAILABLE` | 503 | 保留原文并创建简单任务 |
| `ATTACHMENT_INVALID` | 415/413 | 更换文件 |
