# 育儿协作台功能板块设计交接清单

> 用途：给设计同事对接移动端原型与后续视觉替换。  
> 当前阶段只锁定功能结构、页面顺序、模块内容和交互入口，不锁定最终设计语言。

---

## 0. 全局结构

### 0.1 产品主闭环

用户核心路径：

1. 上传或选择育儿护理视频。
2. Agent 解析视频，生成结构化护理知识。
3. 用户确认知识点、风险边界和适用场景。
4. 系统结合家庭成员画像，生成待办任务。
5. 任务进入待办和日历。
6. 家庭成员按负责人、时间、协作者执行。
7. 完成记录进入贡献热力图。
8. 执行反馈沉淀为家庭经验和 RAG 知识库。
9. 家庭状态变化时，Agent 局部重新编排任务。

核心设计原则：

- 每个任务都要回答四件事：谁负责、什么时候做、为什么这样做、依据是什么。
- 每个任务都要能追溯到来源：视频时间点、结构化步骤、注意事项、风险等级。
- 每次完成都不只是打勾，还要沉淀为家庭经验。
- Agent 不是单独页面工具，而是贯穿待办、日历、贡献和知识库的协作入口。

### 0.2 全局导航

底部固定四个 Tab：

1. 待办
2. 日历
3. 贡献
4. Agent

右侧固定圆形小助手按钮：

- 位置：底部导航右侧，独立圆形按钮。
- 功能：随时唤起 Agent。
- 入口能力：
  - 即时问答
  - 上传视频更新 RAG
  - 根据当前页面内容重新编排任务
  - 解释某个任务的来源和分配原因

### 0.3 全局数据对象

设计需要在页面中持续体现这些对象：

- 家庭组：一个家庭空间，多成员可见。
- 成员账号：妈妈、爸爸、老人、月嫂等。
- 成员画像：身份、经验、可用时间、限制、偏好。
- 宝宝状态：刚喂奶、睡眠中、哭闹、洗澡后、异常需关注等。
- 护理知识：来自视频/RAG 的结构化步骤。
- 任务：负责人、协作者、时间、状态、风险、来源。
- 完成记录：完成者、完成时间、备注、是否沉淀经验。
- 贡献记录：按任务完成数统计。
- 成就徽章：先预留位置，后续根据规则解锁。

## 1. Page 1：待办 To Do List

### 1.1 页面定位

这是家庭成员每天最常用的执行页。核心目标是让用户快速知道：

- 现在有哪些事要做。
- 哪些任务优先级最高。
- 谁负责、谁协助。
- 什么时候做。
- 当前宝宝状态是否影响任务。
- 任务依据来自哪里。
- 完成后是否进入家庭经验。

### 1.2 页面模块顺序

1. 顶部状态区
2. 页面标题：待办
3. 顶部操作区
4. 优先级分组
5. 待办事项列表
6. 任务卡片
7. 经验沉淀提示
8. 添加任务入口
9. 底部导航 + 圆形 Agent 按钮

### 1.3 顶部状态区

需要展示：

- 当前家庭组或用户入口。
- Pro/升级入口可先保留占位。
- 更多操作按钮。
- 新增任务按钮。

可选操作：

- 切换家庭组
- 查看成员
- 设置
- 手动新增任务
- 让 Agent 生成任务

### 1.4 优先级分组

分组：

- 高
- 中
- 低

每组需要包含：

- 分组标签
- 展开/收起状态
- 快速新增按钮
- 当前任务数量，可后续补

设计注意：

- 这里先保持轻量，不做复杂看板。
- 分组不应抢走任务卡片的视觉重心。

### 1.5 任务卡片字段

每张任务卡必须预留这些信息：

- 任务标题
- 优先级
- 负责人
- 协作者
- 计划时间
- 预计耗时
- 宝宝状态
- 来源依据
- 风险等级
- 当前状态
- 完成按钮

推荐卡片信息结构：

```text
任务标题
负责人 · 时间 · 协作者
来源：视频 02:10 · 宝宝状态：刚喂奶
风险/注意事项标签
[查看步骤] [改派] [完成]
```

任务状态：

- 待办
- 即将开始
- 进行中
- 已完成
- 已跳过
- 受影响
- 已取消

风险等级：

- 低风险：可执行
- 需确认：执行前需要主要照护者确认
- 医疗风险：不生成执行任务，只展示风险提示

### 1.6 任务详情底部抽屉

点击任务卡进入底部抽屉。

抽屉内容：

- 任务标题
- 负责人和协作者
- 计划时间
- 预计耗时
- 分配原因
- 操作步骤
- 注意事项
- 来源视频时间点
- RAG 引用来源
- 宝宝状态
- 风险提示
- 操作按钮

操作按钮：

- 标记完成
- 报告变化
- 改派
- 调整时间
- 查看来源片段
- 沉淀为经验

### 1.7 完成后沉淀

完成任务后需要出现轻量反馈：

- 是否完成顺利
- 是否有异常
- 是否需要记录为家庭经验
- 是否反馈给 Agent 优化下次安排

设计可以先做成小卡片或完成弹层。

## 2. Page 2：日历 Calendar

### 2.1 页面定位

日历页负责展示家庭任务排期，同时保留普通日历能力。它不是单纯日程表，而是“任务时间安排 + Agent 动态调整”的页面。

### 2.2 页面模块顺序

1. 顶部 Agent 同步提示
2. 日期标题
3. 周视图日期条
4. 日历操作卡片
5. 时间段分组
6. 日程/任务卡片
7. 受影响任务提示
8. 底部导航 + 圆形 Agent 按钮

### 2.3 顶部 Agent 同步提示

展示内容：

- Agent 已同步今日任务
- Agent 已根据家庭日历更新排期
- 有冲突时提示“发现冲突”

状态：

- 已同步
- 同步中
- 有冲突
- 需要用户确认

### 2.4 日期区

需要展示：

- 星期
- 年月
- 左右切换
- 周视图日期条
- 当前日期高亮

基础能力：

- 切换日期
- 查看某天任务
- 返回今天

### 2.5 日历操作卡片

建议两个入口：

1. 导入日历
   - 家庭共享日历
   - 手机系统日历
   - 微信/其他日程后续可预留

2. Agent 排期
   - 根据成员可用时间安排
   - 根据宝宝状态调整
   - 根据任务风险和优先级排序

### 2.6 时间段分组

推荐分组：

- 随时
- 上午
- 白天
- 晚上
- 夜间

每个分组展示：

- 分组名称
- 任务数量
- 展开/收起
- 快速新增

### 2.7 日程/任务卡片字段

字段：

- 任务标题
- 负责人
- 开始时间
- 预计耗时
- 来源时间点
- 是否由 Agent 排期
- 是否与日历冲突
- 完成状态

卡片操作：

- 完成
- 改时间
- 改负责人
- 查看步骤
- 让 Agent 重新安排

### 2.8 Agent 联动规则

需要预留这些场景：

- 成员不可用：如爸爸加班。
- 宝宝状态变化：如刚喂奶、睡着、哭闹。
- 任务延误：当前任务未完成影响后续。
- 日历冲突：家庭成员已有安排。
- 用户手动锁定：已确认任务不被 Agent 自动改动。

## 3. Page 3：贡献热力图 + 成就

### 3.1 页面定位

贡献页用于展示家庭协作成果，不做社交化排名为主，而是帮助家庭看到：

- 最近完成了多少护理任务。
- 谁参与了照护。
- 哪些习惯正在形成。
- 哪些成就未来可以解锁。

### 3.2 页面模块顺序

1. 顶部用户/家庭入口
2. 总贡献标题
3. GitHub 风格贡献热力图
4. 成就徽章区
5. 家庭贡献排行
6. 阶段复盘入口
7. 底部导航 + 圆形 Agent 按钮

### 3.3 贡献统计口径

当前阶段主要统计：

- 完成任务数

后续可扩展：

- 连续完成天数
- 上传视频数量
- 沉淀经验数量
- 成员协作次数
- Agent 调整被采纳次数

### 3.4 热力图结构

参考 GitHub：

- 标题：过去一年完成贡献数
- 横向月份
- 纵向星期
- 每个格子代表一天
- 颜色深浅代表完成任务数量
- Less / More 图例

点击某一天后展示：

- 当日完成任务数
- 完成成员
- 关键任务
- 是否有沉淀经验

### 3.5 成就徽章区

当前先预留位置，不要求完整规则。

建议首批徽章：

- 日间连胜
- 完工
- 首次上传视频
- 家庭协作
- 知识沉淀
- 夜间守护
- 连续 7 天
- 低风险任务全完成

每个徽章字段：

- 徽章名称
- 当前进度
- 解锁条件
- 是否已解锁
- 解锁时间

展示状态：

- 已解锁
- 进行中
- 未解锁
- 锁定占位

### 3.6 家庭贡献排行

展示：

- 成员头像/名称
- 完成任务数
- 协作任务数
- 最近活跃时间

设计注意：

- 排行不是为了制造压力，而是表达家庭协作参与度。
- 可以弱化竞争感，突出“共同照护”。

## 4. Page 4：RAG 育儿协作 Agent

### 4.1 页面定位

Agent 页是知识与任务的入口。它需要同时承担：

- 育儿知识即时问答
- 上传视频更新 RAG
- 从知识生成待办
- 解释任务依据
- 动态调整排期

### 4.2 页面模块顺序

1. Agent 头像/标题
2. 能力说明
3. 问答/视频入库分段切换
4. 即时问答卡片
5. 上传视频更新 RAG
6. 入库流程展示
7. 安全边界提示
8. 最近知识/来源记录
9. 底部导航 + 圆形 Agent 按钮

### 4.3 即时问答

用户可问：

- 今天有什么计划？
- 这个任务为什么分配给爸爸？
- 宝宝刚喂完奶，现在适合做什么？
- 某个护理动作需要注意什么？
- 哪些任务还没完成？
- 如果爸爸今晚加班，怎么调整？

回答需要展示：

- 简短结论
- 依据来源
- 相关任务
- 可执行建议
- 风险提示

回答结构：

```text
结论
依据：宝宝状态 / 视频时间点 / 家庭画像 / 日历
建议操作
可选按钮：[生成待办] [调整日历] [查看来源]
```

### 4.4 上传视频更新 RAG

入口：

- 上传本地视频
- 选择已有视频
- 粘贴/导入转写结果
- 后续可接抖音链接解析

上传后流程：

1. 校验格式、大小、时长。
2. 调用视频理解或音频转写。
3. 生成结构化护理步骤。
4. 标记风险等级。
5. 用户确认。
6. 写入 RAG 知识库。
7. 生成或更新待办任务。

上传状态：

- 未上传
- 上传中
- 解析中
- 需要确认
- 已入库
- 解析失败

### 4.5 知识卡片字段

每条入库知识需要展示：

- 视频主题
- 适用场景
- 结构化步骤数
- 所需物品
- 注意事项
- 不确定项
- 风险等级
- 来源时间点
- 入库时间
- 创建人

### 4.6 安全边界

必须有明确提示：

- 涉及疾病诊断、药物剂量、异常症状判断或危险操作，不生成执行任务。
- 此类内容只作为风险提示，并建议咨询专业医护。
- Agent 的调整必须经过用户确认后才写入待办或日历。

设计建议：

- 安全提示不要做成吓人的大警告，但要清晰可见。
- 在任务卡、Agent 回答、视频解析确认页都需要出现风险提示入口。

## 8. 当前四页开发优先顺序

推荐开发顺序：

1. Agent 视频解析与知识入库
2. 家庭组与成员画像
3. 待办任务生成与执行
4. 日历排期与 Agent 调整
5. 完成记录与贡献热力图
6. 成就徽章规则
7. 复盘与长期家庭经验库

原因：

- Agent 和视频解析是差异化能力。
- 家庭成员画像决定任务如何分配。
- 待办是最核心执行界面。
- 日历是待办的时间视图。
- 贡献和徽章依赖完成记录。
